field doctor - success toast & restart modal components

Toast and restart modal in mobile + desktop context. All use --fd- design tokens.

how to use this page

  • Section 1 — toast shows the "menu saved" toast in mobile and desktop frames. Each variant has a replay button to re-trigger the animation and a collapsible implementation spec with tokens, classes, and behaviour notes.
  • Section 2 — restart modal shows the confirmation modal with backdrop. Click replay to reopen the modal. Clicking "restart deliveries" dismisses the modal and reveals a welcome-back toast; the toast auto-dismisses after 10 s or can be closed manually.
  • Every component is self-contained — copy the HTML and matching .fd- CSS classes into production. Design tokens are defined as CSS custom properties at the top of this file.

CSS classes

  • .fd-toast — base container (flex row)
  • .fd-toast--top — anchored to top of phone frame (absolute, full width)
  • .fd-toast--exit — exit animation modifier
  • .fd-toast--hidden — display none after exit
  • .fd-toast__icon — 24×24 SVG container
  • .fd-toast__text — message label (flex: 1)
  • .fd-toast__dismiss — close button (32×32 tap target)

Animations

  • Entry: fd-slide-down 0.35s cubic-bezier(0.22, 1, 0.36, 1) — translateY(-100%) → 0
  • Exit: fd-slide-up 0.6s cubic-bezier(0.4, 0, 0.2, 1) — translateY(0) → -100%
  • Checkmark: fd-check-draw 0.4s ease-out 0.25s — stroke-dashoffset 22 → 0

Behaviour

  • Auto-dismisses after 10s (adds --exit, then --hidden on animationend)
  • Dismiss button triggers same exit flow immediately
  • Checkmark SVG is cloned on replay to restart stroke animation

Responsive

  • Mobile: full-width inside phone frame, no border-radius, no left border
  • Desktop: inset toast within .fd-desktop__main, inherits container width

Accessibility

  • role="status" + aria-live="polite" for screen reader announcement
  • Dismiss button uses × character (consider aria-label="dismiss")

CSS classes

  • .fd-modal — card container (white, overflow hidden)
  • .fd-modal__head — title area
  • .fd-modal__title — heading (h4, lowercase)
  • .fd-modal__sub — subtitle paragraph
  • .fd-modal__body — content area
  • .fd-modal__actions — button stack (flex column, gap 4px)
  • .fd-phone__overlay / .fd-desktop__overlay — backdrop + centring
  • --exit modifier — triggers exit animations
  • --hidden modifier — display none after exit

Sub-components

  • .fd-offer — pink highlight strip (--fd-pink bg, bold text)
  • .fd-detail-block — summary table (--fd-offwhite bg, 14px 16px padding)
  • .fd-detail-row — flex row with label + value
  • .fd-detail-row__value--discount — red (--fd-red) for discount values
  • .fd-divider — 1px #eee border
  • .fd-total-row — total with large price (--fd-text-xl, weight 700)

Buttons

  • Primary: --fd-gold bg, black text, --fd-forest border, 13px 24px padding
  • Secondary: transparent, --fd-text-secondary, no border, 12px 24px

Animations

  • Entry: fd-pop-in 0.6s cubic-bezier(0.4, 0, 0.2, 1) — applies to .fd-modal only (backdrop appears instantly)
  • Exit backdrop: fd-fade-out 0.6s — opacity 1 → 0 on overlay
  • Exit card: fd-pop-out 0.6s — scale(1) → scale(0.95) + opacity fade on .fd-modal
  • After animationend on overlay: --hidden class applied

Behaviour

  • "not now" button triggers exit sequence

Responsive

  • Mobile: overlay padding --fd-space-base (16px), modal fills available width
  • Desktop: overlay padding 32px, modal capped at max-width: 400px
